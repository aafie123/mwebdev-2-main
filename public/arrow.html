Arrow functies

In Javascript IE6 kan ook gebruik gemaakt worden van zgn. fat arrow functions. Een fat arrow of double arrow is een combinatie van het is-gelijk teken en het groter-dan teken: =>. In principe betreft het in dat geval altijd een anonieme functie.
We bekijken achtereenvolgens de verschillende vormen die we tegen kunnen komen, maar beginnen met de klasieke anonieme functie:

const som = function(getal1, getal2){ 
    return getal1 + getal2; 
}
document.write('De som is ' + som(2, 5)); // → De som is 7

Er worden 2 argumenten gebruik in de anonieme functie, die toegekend wordt aan de variabele som. Daarna roepen we deze variabele aan binnen een document.write en geven als argumenten de getallen 2 en 5 mee. De uitkomst is het getal 7.
Ditzelfde kunnen we ook schrijven met de arrow functie:

const som = (getal1, getal2) => { 
    return getal1 + getal2; 
} 
document.write('De som is ' + som(2, 5)); // → De som is 7
  1. Wat gebeurt hier?
  2. Het keyword function is komen te vervallen.
  3. Daarentegen is er na de haakjes met de argumenten een fat arrow verschenen.
  4. Daarna volgt de code van de functie tussen accolades.
  5. De functie met accolades moet gebruik maken van het keyword return om een waarde te retourneren, die in de variabele som geplaatst kan worden.

Dan is er ook een verkorte variant:

const som = (getal1, getal2) => getal1 + getal2;
document.write('De som is ' + som(2, 5)); // → De som is 7
  1. En wat zien we nu?
  2. De accolades zijn komen te vervallen.
    Dat is toegestaan als er maar 1 expressie gebruikt wordt in de functie.
  3. Ook het return keyword mag in dat geval worden weggelaten.

Er zijn nog enkele vormen, maar die laten we hier buiten beschouwing. Dit zijn de meest voorkomende vormen van de arrow functie. Het is belangrijk om deze vormen te kennen, zodat je het kunt lezen als je het tegenkomt.
Als je functies gebruikt, of dat nu de klassieke of de arrow functions zijn, het is altijd het belangrijkste dat functies duidelijk en goed te lezen zijn. Dat een arrow functie korter is is niet essentiëel.
